# TinyCraft

TinyCraft - небольшая Java/LWJGL voxel-песочница в стиле Minecraft. В проекте есть чанковый мир, биомы, горы, пещеры, шахты, деревни, мобы, инвентарь, крафт, печки, жидкости, команды и первый экспериментальный LAN-мультиплеер без внешних аккаунтов.

Текущая версия документации описывает `v0.2 Snapshot 7`.

## Быстрый старт

1. Установите Java 8 или новее.
2. Скачайте сборку `TinyCraft-v0.2-snapshot7-windows.zip` со страницы релизов.
3. Распакуйте архив в отдельную папку.
4. Запустите `run-game.bat`.

Для запуска из исходников:

```powershell
javac -encoding UTF-8 --release 8 -cp "lib/*" -d out *.java
.\run-game.bat
```

Или вручную:

```powershell
java -cp "out;lib/*" TinyCraft
```

## Возможности

- Чанковый voxel-мир с прогрузкой колонок и туманом непрогруженных чанков.
- Рельеф, горы, пещеры, руды, реки, океаны и биомы.
- Деревни, фермы, дома, шахты и простые структуры.
- Базовое освещение, прозрачные блоки, вода и лава.
- Инвентарь, хотбар, крафт, печки, сундуки и верстак.
- Выживание, творческий режим, режим наблюдателя, здоровье и голод.
- Мобы, дроп, яйца спавна и простой бой.
- Внутриигровой чат, команды и debug overlay.
- MVP LAN-мультиплеера через Direct IP.

## Мультиплеер Snapshot 7

Snapshot 7 добавляет первый рабочий MVP мультиплеера для локальной сети и тестов на одном ПК. Это не публичная серверная инфраструктура, а прямое TCP-подключение между двумя копиями игры.

### Архитектура

- Транспорт: чистый TCP без сторонних сетевых библиотек.
- Формат пакетов: `int length` + `byte packetId` + payload через `DataInputStream` и `DataOutputStream`.
- Версия протокола: `MAGIC = TCMP`, `VERSION = 1`.
- Порт по умолчанию: `25566`.
- Профиль игрока: локальный `profile.properties` с UUID и ником.
- Handshake: клиент отправляет `HELLO`, хост отвечает `WELCOME`.
- Хост: интегрированный сервер в отдельном потоке, авторитетен по миру, блокам, чату, мобам, дропу и времени.
- Клиент: mirror-режим, получает состояние от хоста, запрашивает чанки и отправляет действия с блоками.
- Движение: клиент использует локальное предсказание и регулярно отправляет `PLAYER_STATE`.
- Мир: клиент получает `CHUNK_DATA`, `BLOCK_UPDATE`, `WORLD_TIME`, snapshots мобов и выпавших предметов.
- UI: добавлен экран "Мультиплеер" и подменю открытия текущего мира для LAN.

### Основные пакеты

| ID | Пакет | Назначение |
| --- | --- | --- |
| 1 | `HELLO` | Приветствие клиента, версия протокола, UUID и ник. |
| 2 | `WELCOME` | Ответ хоста: seed, terrain preset, позиция спавна и время мира. |
| 4 | `PLAYER_STATE` | Позиция, поворот, выбранный предмет и флаги игрока. |
| 6 | `CHAT` | Сообщения чата. |
| 7 | `CHUNK_REQUEST` | Запрос клиентом колонки чанка. |
| 8 | `CHUNK_DATA` | Данные колонки мира от хоста. |
| 9 | `BLOCK_ACTION` | Запрос клиента на ломание или установку блока. |
| 10 | `BLOCK_UPDATE` | Авторитетное изменение блока от хоста. |
| 11 | `WORLD_TIME` | Синхронизация времени мира. |
| 12 | `MOB_SNAPSHOT` | Снимок состояния мобов. |
| 13 | `DROPPED_ITEM_SNAPSHOT` | Снимок выпавших предметов. |
| 14 | `DISCONNECT` | Отключение с причиной. |

## Ограничения мультиплеера

- Только Direct IP/LAN. Нет браузера серверов, relay, NAT traversal и публичного matchmaking.
- Нет внешних аккаунтов и авторизации. UUID и ник берутся из локального `profile.properties`.
- Нет dedicated server: хостом является один из игроков с открытым миром.
- Нет anti-cheat. Хост авторитетен по миру, но клиентским данным пока доверяют больше, чем нужно для публичной игры.
- Нет синхронизации содержимого сундуков и других сложных контейнеров.
- Нет полноценной синхронизации всех UI-состояний, рецептов, прогресса и приватных инвентарных действий.
- При подключении с одинаковым UUID хост отклоняет клиента как дубликат.
- Интернет-игра возможна только при ручном пробросе порта, VPN или другой сетевой настройке.

## Как протестировать мультиплеер

### Тест на одном ПК через два окна

1. Скомпилируйте проект:

   ```powershell
   javac -encoding UTF-8 --release 8 -cp "lib/*" -d out *.java
   ```

2. Откройте два отдельных окна PowerShell в папке проекта.
3. В первом окне запустите игру:

   ```powershell
   .\run-game.bat
   ```

4. Создайте или загрузите одиночный мир.
5. Откройте меню, выберите "Мультиплеер", затем "Открыть мир для LAN".
6. Оставьте порт `25566` или укажите свой.
7. Нажмите "Открыть мир для LAN". В статусе мультиплеера должен появиться текст вида `Hosting on port 25566`.
8. Во втором окне запустите вторую копию игры:

   ```powershell
   .\run-game.bat
   ```

9. На экране "Мультиплеер" укажите:

   - Host: `127.0.0.1`
   - Port: `25566`
   - Nick: любой ник, отличающийся от первого окна

10. Нажмите подключение по Direct IP.
11. На клиенте статус должен пройти через `Connecting to 127.0.0.1:25566` и стать `Connected`.
12. На хосте в чате должно появиться сообщение вида `<ник> joined the world.`.
13. Подвигайтесь клиентом: на хосте должна появиться удаленная модель игрока.
14. Сломайте или поставьте блок клиентом: изменение должно примениться на хосте и вернуться клиенту как авторитетный `BLOCK_UPDATE`.
15. Напишите сообщение в чат с клиента: оно должно появиться у обоих игроков.

Если обе копии игры используют один и тот же `profile.properties`, подключение может быть отклонено как `Duplicate player uuid.`. Для теста на одном ПК проще запускать вторую копию из отдельной папки проекта или временно удалить/изменить `profile.properties` во второй папке, чтобы игра создала новый UUID.

### Тест в локальной сети

1. На хосте запустите игру, загрузите мир и откройте его для LAN.
2. Разрешите Java доступ к частным сетям в Windows Firewall, если появится системный запрос.
3. Узнайте IP хоста в локальной сети:

   ```powershell
   ipconfig
   ```

   Обычно нужен адрес вида `192.168.x.x` или `10.x.x.x`.

4. На втором ПК запустите игру.
5. В меню "Мультиплеер" укажите IP хоста и порт `25566`.
6. Подключитесь и проверьте статус `Connected`, сообщение `joined the world`, прогрузку мира, чат и установку/ломание блоков.

Если клиент не подключается, проверьте, что оба ПК в одной сети, порт совпадает, firewall не блокирует Java, а хост действительно показывает статус `Hosting on port 25566`.

## На какие логи смотреть

В Snapshot 7 сетевой слой не печатает подробный лог каждого пакета в консоль. Основные признаки успешной работы видны в игровом статусе и чате:

- `Hosting on port 25566` - хост поднял TCP-сервер.
- `Connecting to <ip>:25566` - клиент начал подключение.
- `Connected` - клиент получил `WELCOME`, handshake завершен.
- `<ник> joined the world.` - хост принял игрока и разослал чат-событие.
- `Disconnected: ...`, `Host failed: ...`, `Host accept failed: ...`, `Chunk sync failed: ...` - ошибки подключения или синхронизации.

В консоли дополнительно могут появляться профильные строки мира:

- `World profile: ...` - первичная генерация мира.
- `VoxelWorld profile: loaded=... pending=... submitted=... integrated=...` - стриминг колонок/чанков. На клиенте рост `loaded` и ненулевые `integrated` после подключения обычно подтверждают, что `CHUNK_REQUEST` и `CHUNK_DATA` проходят.

## Управление

- `WASD` - движение
- `Space` - прыжок
- `Shift` - присесть
- `Ctrl` - бег
- Левая кнопка мыши - атака или ломание блока
- Правая кнопка мыши - взаимодействие или установка блока
- `T` - чат
- `E` - инвентарь
- `Esc` - меню паузы
- `F1` - скрыть или показать интерфейс
- `F3` - debug overlay
- `F4` - переключение режима игры
- `F5` - вид от третьего лица
- `1`-`9` - выбор слота хотбара

## Команды

Команды вводятся в игровом чате.

| Команда | Что делает |
| --- | --- |
| `/tp <x> <y> <z>` | Телепортирует игрока по координатам. |
| `/time set day` | Устанавливает день. |
| `/time set night` | Устанавливает ночь. |
| `/gamemode creative` | Включает творческий режим. |
| `/gamemode survival` | Включает режим выживания. |
| `/gamemode spectator` | Включает режим наблюдателя. |
| `/clear` | Очищает инвентарь. |
| `/say <сообщение>` | Выводит сообщение от сервера. |
| `/give <id> <количество>` | Выдает предмет или блок по ID. |
| `/spawnzombie` | Спавнит зомби рядом с игроком. |
| `/seed` | Показывает seed мира. |
| `/locate village` | Ищет ближайшую деревню. |
| `/locate mineshaft` | Ищет ближайшую шахту. |
| `/locate biome <название>` | Ищет биом. |
| `/place structure list` | Показывает список доступных структур. |
| `/place structure <name> [rotation]` | Ставит структуру рядом с игроком. |
| `/whereami` | Показывает текущую debug-позицию. |
| `/probe <x> <z>` | Показывает terrain/debug-информацию по координатам. |
| `/terrain <x> <z>` | То же, что `/probe`. |
| `/heighttest` | Запускает debug-проверку высот. |
| `/blockinfo` | Показывает информацию о блоке под прицелом. |
