# FAQ

## Что это за проект?

TinyCraft / TinyMinecraft - маленькая Java/LWJGL voxel-песочница в стиле Minecraft. В проекте есть чанковый мир, биомы, горы, пещеры, шахты, деревни, мобы, инвентарь, крафт, сундуки, печки, жидкости, команды и экспериментальный LAN-мультиплеер.

## Какая версия сейчас актуальная?

Актуальный пререлиз разработки - `v0.2 Snapshot 7`.

Последний стабильный релиз на GitHub может все еще отображаться как `TinyMinecraft v0.1`, потому что Snapshot 7 опубликован как prerelease.

## Где скачать игру?

Откройте страницу релизов:

[https://github.com/vakisnn-gd/TinyMinecraft/releases](https://github.com/vakisnn-gd/TinyMinecraft/releases)

Для Snapshot 7 используйте релиз:

- `v0.2 Snapshot 7`
- tag: `v0.2-snapshot7`

Если у релиза есть готовый `.zip`, скачивайте именно его. `Source code` - это архив исходников от GitHub, он больше подходит для разработки.

## Какая Java нужна?

Для Snapshot 7 нужна Java 8 или новее.

Проект собирается командой с `--release 8`, поэтому он совместим с Java 8 runtime. Запускать игру можно и на более свежем JDK. Предупреждения Java о том, что `source/target 8` устареют в будущем, не являются ошибкой.

Проверить Java:

```powershell
java -version
```

## Как собрать и запустить из исходников?

```powershell
javac -encoding UTF-8 --release 8 -cp "lib/*" -d out *.java
java -cp "out;lib/*" TinyCraft
```

На Windows также можно запустить:

```powershell
.\run-game.bat
```

## Поддерживается ли мультиплеер?

Да, начиная с `v0.2 Snapshot 7` есть MVP LAN-мультиплеера.

Что работает:

- Direct IP / LAN подключение.
- Хостинг мира на порту `25566`.
- Подключение клиента к `127.0.0.1` на одном ПК.
- Синхронизация чанков.
- Чат, включая русский текст.
- Установка и ломание блоков.
- Модели удаленных игроков.
- Базовый PvP.

Что пока не готово:

- Нет публичного списка серверов.
- Нет dedicated server.
- Нет внешних аккаунтов.
- Нет anti-cheat.
- Нет синхронизации сундуков и сложных контейнеров.
- Никнеймы над игроками временно отключены.

## Как открыть мир для LAN?

1. Запустите игру.
2. Создайте или загрузите одиночный мир.
3. Откройте меню паузы.
4. Перейдите в "Мультиплеер".
5. Нажмите "Открыть мир".
6. Оставьте порт `25566` или укажите свой.
7. Дождитесь статуса `Hosting on port 25566`.

## Как подключиться к хосту на том же ПК?

На клиенте в меню "Мультиплеер" укажите:

- IP: `127.0.0.1`
- Port: `25566`
- Nick: любой ник, отличный от хоста

Если обе копии игры используют один и тот же `profile.properties`, хост отклонит подключение как `Duplicate player uuid.`. Для теста на одном ПК запускайте клиента из отдельной рабочей папки или создайте ему отдельный `profile.properties`.

## Как подключиться по локальной сети?

На хосте:

```powershell
ipconfig
```

Найдите IPv4-адрес вида `192.168.x.x` или `10.x.x.x`.

На клиентском ПК введите этот IP и порт `25566`. Если подключение не идет, проверьте Windows Firewall и убедитесь, что оба ПК находятся в одной сети.

## Почему в GitHub есть ветка `main`?

`main` - главная ветка репозитория. Именно ее GitHub показывает по умолчанию на странице проекта.

Рабочие ветки вроде `codex/snapshot-5` используются для подготовки изменений. Когда Snapshot готов, изменения нужно слить в `main`, чтобы главная страница репозитория показывала актуальный README, файлы и скриншоты.

## Что такое `profile.properties`?

Это локальный профиль игрока:

- UUID
- ник

Файл создается автоматически и не должен попадать в GitHub. Он добавлен в `.gitignore`.

## Что делать, если Windows предупреждает про `.bat` файл?

`run-game.bat` - обычный запускатель для Windows. Он вызывает Java с нужным classpath и библиотеками из папки `lib/`.

## Какие команды есть в игре?

Основные команды:

```text
/tp <x> <y> <z>
/time set <day|night>
/gamemode <creative|survival|spectator>
/clear
/say <сообщение>
/give <id> <количество>
/spawnzombie
/seed
/locate village
/locate mineshaft
/locate biome <название>
/place structure list
/place structure <name> [rotation]
/whereami
/probe <x> <z>
/terrain <x> <z>
/heighttest
/blockinfo
```

Подробнее см. [README](README.md).
